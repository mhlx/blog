评论标签

<data name="commentPage" moduleType="" moduleId="" mode="" asc="" currentPage="" pageSize=""/>

可接受的标签属性

属性名 类型 是否必须 默认 说明
moduleType USERPAGE|ARTICLE|NEWS 是 模块类型
moduleId Integer 是 模块ID
currentPage Integer 否 0 当前页码
pageSize Integer 否 10 分页数目，不会超过设置的最大分页数目
mode LIST|TREE 否 LIST 展现方式(列表|嵌套)
asc Boolean 否 true 如果为true，评论按照时间从小到大排序，否则逆向排序

结果返回

无论是否查询到结果，都将会返回一个不为null的CommentPageResult对象，该对象继承自PageResult<Comment>，基础可访问属性请见PageResult默认属性，除此之外， 它还有一个commentConfig属性，查看commentConfig可供访问的属性

根据CommentMode的不同，Comment的parent可供访问属性也不同，但都有以下的基本可访问属性

属性名 类型 是否可能为null 说明
id Integer 否 评论ID
content String 否 评论内容(HTML格式)
commentModule CommentModule 否 评论模块， 查看可供访问属性
commentDate Timestamp 否 评论日期
status enum 否 评论状态，NORMAL(正常)|CHECK(审核)
nickname String 否 评论人昵称
email String 是 评论人邮箱
ip String 否 评论人ip
admin Boolean 否 是否是管理员评论
gravatar String 是 评论人头像，该值为邮箱的md5值，如果评论人没有输入邮箱，那么这个值为null
website String 是 评论人的站点地址
editor enum 否 评论编辑器，MD(markdown)|HTML(富文本)，改编辑器只用于标记评论时的编辑方式，实际输出已经被转为html文本
url String 否 CommentModule的实际地址

当CommentMode为TREE的时候，Comment的parent可访问属性如下：

属性名 类型 是否可能为null 说明
id Integer 否 评论ID

当CommentMode为LIST的时候，Comment的parent可访问属性如下：

属性名 类型 是否可能为null 说明
id Integer 否 评论ID
commentModule CommentModule 否 评论模块， 查看可供访问属性
nickname String 否 评论人昵称
email String 是 评论人邮箱
ip String 否 评论人ip
admin Boolean 否 是否是管理员评论
gravatar String 是 评论人头像，该值为邮箱的md5值，如果评论人没有输入邮箱，那么这个值为null
url String 否 CommentModule的实际地址

CommentConfig属性

属性名 类型 是否可能为null 说明
editor enum 否 评论编辑器，可能值为HTML(富文本)和MD(markdown)
check Boolean 否 是否审核评论
pageSize Integer 否 评论每页展示数量

CommentModule属性

属性名 类型 是否可能为null 说明
id Integer 否 模块ID
type enum 否 模块类型，USERPAGE|ARTICLE
